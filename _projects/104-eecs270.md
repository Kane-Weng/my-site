<h2 class="section-header learn">Topic 1: Foundations of Hardware Logic</h2>
This section explores the bridge between physical signals and logical certainty. By manipulating primitive gates in Verilog, I developed the building blocks of digital systems, from simple data routing to complex arithmetic units.

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #1: Data Routing & Selection</div>
      <div class="summary-tags">
        <span class="lang-tag">Verilog</span>

        <span class="topic-tag">Multiplexers (MUX)</span>

      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Developing the foundational understanding of hardware abstraction by building a 7-bit data selector to route signals between high-speed input buses and output displays.</li>
      <li><strong>Build:</strong> Architected a modular 2-to-1 Multiplexer (MUX) using primitive logic gates (<code>and</code>, <code>or</code>, <code>not</code>) to manage 7-bit parallel data streams.</li>
      <li><strong>Functionality:</strong> Integrated the selector with physical FPGA hardware, mapping slide switches to data inputs and pushbuttons to control signals, with real-time feedback on active-low 7-segment displays.</li>
    </ul>
  </div>
</details>

      <li><strong>Build:</strong> Implemented specific gate-level delays (rise/fall times) to analyze signal "drift" and built a <strong>Ring Oscillator</strong> using a chain of $N$ inverters with feedback.</li>
      <li><strong>Functionality:</strong> Conducted timing simulations to observe how gate delays impact system stability, a crucial step for preventing "glitches" in high-frequency robot controllers.</li>
    </ul>

</details>

<h2 class="section-header teach">Topic 2: Sequential Systems & Architecture</h2>
By introducing memory, clock-synchronized state, and feedback loops, I moved from static logic to dynamic architectural design, culminating in the creation of a sequential processor that coordinates complex arithmetic operations over multiple cycles.

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #5: Finite State Machine (FSM) Design</div>
      <div class="summary-tags">
        <span class="lang-tag">Verilog</span>
        <span class="topic-tag">Sequential Logic</span>
        <span class="topic-tag">Synchronous State Updates</span>
        <span class="topic-tag">Bimodal Branch Prediction</span>
      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Implementing the first "memory-aware" system by designing an Up-Down Counter that changes its output based on both current inputs and previous states.</li>
      <li><strong>Build:</strong> Developed two architectural variations: a <strong>Behavioral FSM</strong> using high-level case statements and a <strong>Structural FSM</strong> using raw Boolean next-state equations.</li>
      <li><strong>Functionality:</strong> Simulated a 2-bit saturated counter (often used in CPU branch predictors), tracking "Strong" vs. "Weak" states triggered by synchronized clock edges.</li>
    </ul>
  </div>
</details>

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #6: Real-Time Traffic Controller (TLC)</div>
      <div class="summary-tags">
        <span class="lang-tag">Verilog</span>
        <span class="topic-tag">Clock Division</span>
        <span class="topic-tag">Nested FSMs</span>
        <span class="topic-tag">Timed Sequence Control</span>
      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Architecting a complex, multi-flow intersection controller that synchronizes four different traffic directions based on real-time sensor polling.</li>
      <li><strong>Build:</strong> Designed a custom <strong>Clock Divider</strong> to step down the 50MHz FPGA clock to human-perceptible intervals and managed state transitions through a 26-state FSM.</li>
      <li><strong>Functionality:</strong> Developed a dual-digit countdown timer and ensured safety-critical transitions (Mandatory Yellow states) across East, West, North, and Side-Road traffic patterns.</li>
    </ul>
  </div>
</details>
