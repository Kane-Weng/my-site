<h2 class="section-header teach">Topic 1: Architecting ROS</h2>
This series focuses on the fundamentals of communication in robotics. By stepping away from pre-built middleware, I explored how data moves between hardware and software, implementing custom IPC protocols, reliable network streaming, and a full-scale publish/subscribe ecosystem from the ground up.

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #1: Keyboard Teleoperation & IPC</div>
      <div class="summary-tags">
        <span class="lang-tag">C/C++</span>
        <span class="topic-tag">POSIX System Calls</span>
        <span class="topic-tag">Linux Signals</span>
      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Building a robust command-line interface to drive the MBot Omni, specifically implementing a custom IPC (Inter-Process Communication) library from scratch to handle data flow between the keyboard and the robot hardware.</li>
      <li><strong>Build:</strong> Developed a custom serialization protocol to marshal 2D twist commands across named pipes (FIFOs).</li>
      <li><strong>Functionality:</strong> Developed two primary nodes: <code>teleop_keyboard</code> for translating WASD/QE inputs into serialized messages and <code>mbot_driver</code> to interface with the MBot Control Board.</li>
    </ul>
  </div>
</details>

<details class="project-accordion">

      <li><strong>Build:</strong> Implemented <code>Server</code> and <code>Client</code> abstractions to handle concurrent connections and thread-safe data handling.</li>
      <li><strong>Functionality:</strong> Hosted a TCP server on the MBot to stream real-time <strong>RPLIDAR A1</strong> scans and odometry to a remote client while receiving position commands.</li>
    </ul>
  </div>
</details>

<details class="project-accordion">
  <summary>
    <div class="summary-container">

      <div class="summary-tags">

      <li><strong>Build:</strong> Architected a central discovery server (<code>RIXHub</code>) to manage node registration and direct peer-to-peer subscriber connections.</li>
      <li><strong>Functionality:</strong> Created a fully modular ecosystem where <code>Publisher</code> and <code>Subscriber</code> classes automate connection logic, allowing nodes to be swapped without system interruption.</li>
    </ul>
  </div>
</details>

<h2 class="section-header learn">Topic 2: Deploying ROS</h2>
With the communication layer established, this topic shifts toward spatial intelligence. These projects involve implementing the mathematical backbone of robot motion—navigating complex transformation trees and using numerical optimization to solve kinematic chains for high-DOF manipulators.

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #4: Forward Kinematics & TF Library</div>
      <div class="summary-tags">
        <span class="lang-tag">C/C++</span>
        <span class="topic-tag">Eigen Geometry</span>
        <span class="topic-tag">JRDF (JSON Robot Description)</span>
        <span class="topic-tag">SLERP interpolation</span>
      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Developing a robust transformation library to manage coordinate frame trees and time-synced spatial relationships between robot links.</li>
      <li><strong>Build:</strong> Created a <code>TransformBuffer</code> for time-aware pose lookups and a <code>FrameGraph</code> to maintain the parent-child tree structure.</li>
      <li><strong>Functionality:</strong> Built a <code>Robot State Publisher</code> that broadcasts the real-time 3D pose of every link by interpolating joint states and navigating the kinematic tree.</li>
    </ul>
  </div>
</details>

<details class="project-accordion">
  <summary>
    <div class="summary-container">
      <div class="project-title">Project #5: Inverse Kinematics (IK)</div>
      <div class="summary-tags">
        <span class="lang-tag">C/C++</span>
        <span class="topic-tag">Geometric Jacobian</span>
        <span class="topic-tag">Moore-Penrose Pseudoinverse</span>
        <span class="topic-tag">Gradient Descent</span>
      </div>
    </div>
  </summary>
  <div class="project-content">
    <ul>
      <li><strong>Objective:</strong> Solving the Goal-to-Joint problem using numerical optimization to allow a robot end-effector to follow 6-DOF targets in Cartesian space.</li>
      <li><strong>Build:</strong> Derived an analytical $6 \times N$ Jacobian matrix and implemented an iterative solver with joint clamping and step-scaling.</li>
      <li><strong>Functionality:</strong> Successfully enabled precise end-effector control for serial manipulators (like the Fetch robot), allowing them to reach arbitrary spatial setpoints via optimized trajectories.</li>
    </ul>
  </div>
</details>
